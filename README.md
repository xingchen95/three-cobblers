# 三个臭皮匠 · Three Cobblers

**三个臭皮匠，顶个诸葛亮。** 把一道难题交给 3 个 Opus「臭皮匠」**各自独立**琢磨、再**互相拍砖**，最后由主窗口 Opus「诸葛亮」整合成一个高于任何单个模型的答案。

这是一个 [Claude Code](https://claude.com/claude-code) skill。它的机制是 **独立，再综合**：同一个问题同时发给三个 Opus 子代理，每个都带 web 搜索和 bash、互相看不到对方的回答；然后主窗口把三份答案碰撞、整合。让一个模型「臭皮匠化」成三个独立视角的关键，是**独立性**——但同一个 Opus 跑三遍，错误是相关的。所以 cobblers 在保持首轮 blind 独立的同时，给三人各注入一行「方法论侧重」（只分叉「怎么推」，**不分配**会带偏结论的「角色」），把发散从「温度抖动」推到「不同推理路径」；真正的角色要到互评轮才登场。

```
   问题
    │
    ▼
 ┌──────────────┐  ① Round 1 · 各自闷头想（blind，互不可见）
 │ 诸葛亮 主窗口  │     判题型，给三人各分一行「方法论侧重」——只分叉「怎么推」
 └──────┬───────┘
        ├──▶ 臭皮匠 A · 第一性原理    各自用 web / bash 研究
        ├──▶ 臭皮匠 B · 类比迁移      各交「自述卡 + 证据卡」
        └──▶ 臭皮匠 C · 逆向工程      （主张 / 来源 / 可信度 高·中·低）
        │
        ▼
 ② 研究状态机（主窗口逐轮维护）
    主张 ✅⚔️❌🔬 · 分歧 · 证据缺口 · 验证任务 · 已拒方案
        │
        ▼  ③ Round 2+ · 互相拍砖
 从 5 角色池选 3（默认 严谨/发散/批判），re-spawn 注入另两家观点 + 角色
        │
        ▼  ④ 动态调度每轮判一次 ↺
 CONTINUE 再吵 · REFOCUS 跑题拉回 · ASK_USER 方向之争问你 · STALLED 停 · CONVERGED 收敛
        │
        ▼
 诸葛亮整合（保留分歧，不强压共识）
 共识 · 矛盾 · 部分覆盖 · 独特洞见 · 盲点 · 观点演化 · 最终答案
```

**一条硬规则：主窗口 Opus 永远是「诸葛亮」（裁判 + 整合者），流水线不可逆。** 臭皮匠子代理没法回头 spawn 主窗口，所以整合永远落在主窗口——真正的理由是**可见性**（只有主窗口能看到全部三份答案）。Quick 模式下它不下场写答案、确是旁观者；但 Discuss 模式下它还维护研究状态机、判收敛、贴标签，是**掌握议程的主编**而非纯旁观者。更要警惕的是：**裁判和三个臭皮匠是同一个 Opus，会本能偏好「表述像自己」的那份**（self-preference），所以整合时按证据强度而非流畅度排序，对自己最认同的那份额外做一次证伪。

## 两种模式

| 模式 | 命令 | 流程 | 何时用 |
| --- | --- | --- | --- |
| **Discuss（主打）** | `/cobblers` | 首轮 blind 独立 → 多轮互评（按题型选角）→ 整合 | 真正需要来回碰撞、互相挑错的难题 |
| **Quick** | `/cobblers-quick` | 3 个 Opus 一次性 blind → 直接整合 | 想要多视角但不必多轮，快且省 |

**Discuss** 是完整形态：第 1 轮三人独立发散（互不可见，各带一行方法论侧重）；第 2 轮起，主窗口**每轮 re-spawn 新臭皮匠**、把「另两家的观点」注入新 prompt，并从 5 角色池按题型选 3 个角色让他们互评、反驳、修正；主窗口当调度器判断每轮走向（硬上限 2–3 轮），最后整合。**Quick** 砍掉互评与多轮，只保留「独立作答 + 方法论分叉 + 证据卡 + 整合」，用三分之一的工序换更快更省。

## 怎么把「同一个模型」逼出不同的思考

这是 cobblers 全部精巧之所在，也是它最该被诚实交代的地方。

三个臭皮匠都是同一个 Opus。同一套权重跑三遍，**温度采样只给你「措辞多样」，不给你「错误去相关」**——遇到模型的知识盲区或对齐偏好，三家会一起、且自信地错向同一处。所以 cobblers 在「同模型」这个硬约束下，用五道工序尽量把「看起来不同」逼成「真的从不同地方想」。**这五项机制整体移植自 [mindforge](#致谢)**（一个多 AI 讨论编排器），但裁掉了它依赖外部服务的部分，保持纯 Opus、零依赖：

1. **首轮方法论分叉**——第一轮仍是 blind 同题，但给三人各注入一行「思考侧重」（第一性原理 / 类比迁移 / 逆向工程 / 数值仿真 / 最小复现…，按题型选）。**只分叉「怎么推」、不分叉「推向哪」**，把发散从「同一吸引盆里的温度抖动」推到「不同推理路径」，又不污染「独立得出共识」的信号价值。
2. **结构化证据卡**——每条关键证据都要标清：**支持/反驳哪条主张 · 来源 · 可信度**（高=一手实验/官方规范，中=厂商文档/二手综述，低=博客/纯记忆）。这把「三家凭记忆一致」和「三家有一手来源还一致」**分开**——前者是共享偏置的合谋，后者才配叫高置信。
3. **5 角色池，按题型选 3**——互评轮才登场。角色池有 **严谨者 / 务实者 / 发散者 / 批判者 / 证据者** 五个（各含「角色说明 / 必须体现 / 输出偏好 / 避免」四段职责），主窗口按题型选 3 个（默认 严谨/发散/批判，**严谨者 + 批判者几乎总在场**以保住「挑错」职能）。同为 Opus，靠角色 prompt 制造视角张力。
4. **研究状态机**——主窗口逐轮维护一块结构化黑板：主张表（带 ✅共识 / ⚔️争议 / ❌已反驳 / 🔬待验证）、分歧、证据缺口、**验证任务**（哪些最小验证待做/已做）、**已拒方案**（防止下一轮又把已经否决的方向重新捡起来吵）。
5. **动态调度器**——每轮互评后，主窗口判一次下一步：**CONTINUE** 继续吵（只 re-spawn 与分歧最相关的角色，省 token）/ **REFOCUS** 跑题了拉回（两次拉不回就升级问你）/ **ASK_USER** 方向之争暂停问你 / **STALLED** 没新增就停 / **CONVERGED** 收敛进整合。

**刻意没搬的部分**（cobblers 是纯 Opus 零依赖，搬了会破坏它的身份）：mindforge 的**真异构多模型**（Claude + GPT + Gemini）、**SQLite 跨会话持久记忆**、**内建联网检索基础设施**。当你需要的恰恰是真异构多样性时，那是 `fusion` 的活，不是 cobblers 的（见下「为什么三个臭皮匠能赢一个」）。

## 讨论怎么进行（Discuss 细节）

1. **Round 1 · 各自闷头想**：主窗口先判题型、给三人各分一行方法论侧重，然后三人 blind 独立作答（互不可见）。每人结尾交一张**自述卡**：核心主张 / **证据卡**（每条证据标 支持反驳哪条主张 · 来源 · 可信度）/ 关键假设 / 待验证 / 方法论增益（这行侧重让我看到了别人可能漏的什么）。
2. **选角**：第 1 轮结束后，主窗口按题型从 5 角色池选定 3 个，映射给甲 / 乙 / 丙，一场讨论内固定。
3. **Round 2 · 互相拍砖**：默认环境不能续接已 spawn 的子代理（`SendMessage` 仅在实验性 agent-teams flag 下可用、默认关），所以主窗口**每轮重新 spawn 三个新臭皮匠**，把「它上一轮答案 + 另两家观点 + 研究状态 + 角色 + 互评议程」注入新 prompt。议程只做三件事——挑对方的隐含假设与反例、把每个争议转成可验证的最小动作、明确哪些已稳哪些仍是候选。
4. **研究状态机**：主窗口逐轮维护 主张表 / 分歧（≤3，结构化）/ 证据缺口 / 验证任务 / 已拒方案 / 开放问题，作为下一轮注入每个臭皮匠的「共享黑板」。
5. **动态调度（每轮判一次）**：在 CONTINUE / REFOCUS / ASK_USER / STALLED / CONVERGED 五个动作里选一个。**收敛有硬门槛**：除了「候选方向成形 + 分歧列清 + 证据缺口标出 + 每个结论有验证路径」这套**格式条件**，还要求一条**实质条件**——至少一条关键主张有**一手来源支持**（真跑过代码 / 查过文献 / 做过计算）；否则就算结构填得再工整，也判为「包装精美的假收敛」，强制再来一轮并要求产出一个真实执行的验证。默认 2 轮、最多 3 轮。
6. **整合**：先校准 self-preference（按**证据卡可信度**而非表述流畅度排序、对最认同那份额外证伪），然后保留分歧不强压共识，每个主张标 ✅共识 / ⚔️争议 / ❌已反驳 / 🔬待验证，并给一节「观点演化」，把**已拒方案**和**执行过的验证及结果**一并纳入审计轨迹。

### 上下文传递模式（互评轮另两家的信息给多少）

子代理之间不能直接通信，互评信息全靠主窗口中转（一个有损瓶颈），所以做成可设置：

- **`full`（默认）**：转发各家**完整 Round 1 全文**——连论证链都给，挑隐含假设最扎实。cobblers 主打高风险题，默认就该给全文。
- **`digest`**：转发各家**自述卡**（含证据卡）——作者本人提炼，省 token。
- 切换：`/cobblers --digest <问题>`，或自然语言说「精简上下文」。**只在低风险、或答案太长怕爆上下文时才退 `digest`**——隐含假设往往就藏在被 digest 压掉的论证链里，默认不该丢。

### 存档讨论（`--save`）

子代理看不到你和主窗口的对话历史——它只读到主窗口塞进 prompt 的那点东西。所以当问题依赖上文（引用之前的方案、之前贴的代码、上传的文件）时，主窗口必须把背景显式带给它。`--save` 把这件事做成文件，顺带留一份存档：

```
/cobblers --save <你的问题>        # 可与 --full 叠加
```

主窗口会建一个 `.cobblers/<时间戳>/`，往里写：

- **`context.md`**：问题 + 背景/代码/文件原文 + 指代消解——三个臭皮匠都读它当背景（解决「子代理看不到对话上下文」）。**只放背景，不放任何人的答案**，否则首轮就不 blind 了。
- **`state.md`**：研究状态机（主张/分歧/证据缺口/验证任务/已拒方案），互评轮的子代理读它。
- **`transcript.md`** + 收尾渲染的 **`transcript.html`**：带每家颜色的辩论全程，给你复盘/分享。

**只有主窗口写这些文件，子代理一律只读**——和 mindforge「共享状态由 server 拥有、AI 不直接碰」同构：零并发竞态，也不破坏「诸葛亮是唯一汇编者」。粒度是「轮时」刷新（每轮结束写一次），不是逐字实时。建议把 `.cobblers/` 加进 `.gitignore`。

## 安装

**macOS / Linux：**

```bash
git clone https://github.com/xingchen95/three-cobblers.git
cd three-cobblers
./install.sh
```

**Windows（PowerShell）：**

```powershell
git clone https://github.com/xingchen95/three-cobblers.git
cd three-cobblers
./install.ps1
```

把 skill 装到 `~/.claude/skills/cobblers`、命令装到 `~/.claude/commands`。重启 Claude Code 或 `/reload-skills` 后生效。

> 改安装位置：bash 用 `CLAUDE_CONFIG_DIR=/path/to/.claude ./install.sh`；PowerShell 用 `$env:CLAUDE_CONFIG_DIR='D:\path\.claude'; ./install.ps1`。

## 用法

三种等价方式：

- **自然语言**（自动触发，默认走 Discuss）：
  > 用三个臭皮匠讨论一下：在 200M 行的 Postgres 生产表上 `ALTER TABLE ADD COLUMN` 安全吗？
- **命令**：
  ```
  /cobblers          我的 JWT 刷新轮换设计有重放漏洞吗？
  /cobblers --full   这个分布式锁实现有没有竞态？        # 互评时把完整论证都给三个臭皮匠
  /cobblers-quick    git push --force-with-lease 在共享分支上真的安全吗？
  ```

每次输出统一结构：**最终答案**在最上面，下面是审计轨迹——**共识 / 矛盾 / 部分覆盖 / 独特洞见 / 盲点**（每个主张带 ✅⚔️❌🔬 状态），Discuss 模式再加一节 **观点演化**（某观点谁先提出 → 被谁拍砖 → 最终存活/修正/放弃），让你看清答案是怎么拼出来的。

## 前提

- **Claude Code**，且会话模型为 **Opus 4.8**——臭皮匠子代理和诸葛亮裁判都继承会话模型；换成别的模型，"Opus" 只是名义上的。
- **零外部依赖**：只用 Opus 子代理（`Agent` 工具），不需要任何外部 CLI、数据库或网络服务，开箱即用。

## 成本与延迟

- **Quick** ≈ 3× 单次回答的 token，跑得和最慢的臭皮匠一样慢。
- **Discuss** ≈ 3–5×（多轮 × 3 子代理）。动态调度器会少花一些——收敛在望时只 re-spawn 与未决分歧最相关的角色，而不是每轮都叫齐三个。

这是刻意的取舍：多花钱，换「不在代价高的地方自信地错」。低风险或简单问题，直接让一个模型答更划算——别为了用而用。

## 为什么三个臭皮匠能赢一个

集成多个独立答案再综合，能稳定胜过单个模型；其中相当一部分提升来自**综合这一步本身**（self-consistency / ensemble 方向上有据，但具体增益幅度依任务而定）。本 repo 现附一个最小评测协议（[BENCHMARK.md](BENCHMARK.md)：单 Opus vs Quick vs Discuss 比准确率），但**尚无跑出的数据**——所有"更强"的说法，在你自己测出来之前都只是方向性判断。「三个臭皮匠」把这套「独立 → 综合」的流程，原地实现在 Claude Code 里。

**但要诚实交代适用边界。** 三个臭皮匠是同一个 Opus，跑三遍得到的是**温度采样的多样性，不是模型的多样性**——它们的错误**高度相关**（共享同一训练盲区、同一对齐偏好）。上面那五道工序（方法论分叉、证据卡、角色分化…）是在**把去相关推到"单模型所能及的极限"**，但推不破共享权重这道墙：所以「独立同意 = 高置信」**只在错误彼此独立时成立**。遇到模型有系统性偏见或知识盲区的题，三家会**齐刷刷错向同一处**，而审计轨迹那套精美结构反而会放大你的信心——**结构化的错误比朴素的错误更危险**。这类题（尤其纯事实、强时效、已知盲区）现在被 skill **硬路由**走——cobblers 会主动建议你转**异构方案**：本机的 `fusion`（Opus + GPT-5.5 + Gemini 真异构 panel）或外部检索，而不是用精美结构包装一个同源答案。cobblers 是 fusion 的「零依赖纯 Opus 轻量版」，主场是**推理 / 设计 / 调试 / 权衡**；**代价是砍掉了最值钱的异构多样性**——这个取舍要清楚。
