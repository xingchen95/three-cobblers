在下面的任务上以 **Discuss 模式** 调用 **cobblers** skill（照 `skills/cobblers/references/discuss.md`，角色池见 `references/roles.md`）。

**先解析上下文模式**：`$ARGUMENTS` 含 `--full` → **full**（互评轮转发另两家完整 Round 1 全文，最完整、token 更高）；含 `--digest` 或不指定 → **digest**（默认，转发各家自产的「自述卡」，省 token、保真）。把标志从问题文本里剥掉再用。

执行流程：
1. **Round 1（blind 独立 + 方法论分叉）**：先一句话判题型，给甲/乙/丙各选一行正交「方法论侧重」（第一性原理 / 类比迁移 / 逆向工程 / 数值仿真 / 最小复现…，只分叉「怎么推」不分叉「推向哪」）。同一条消息 spawn **3 个独立 Opus 子代理**（`Agent`，`subagent_type: general-purpose`，`run_in_background: true`，prompt = 用户任务**逐字原文** + 简短指令 + 那一行方法论侧重，**不分配角色**），各自用 web/bash 研究、给完整自洽中文答案，结尾附**自述卡 + 结构化证据卡**（核心主张 ｜ 每条证据标 支持/反驳哪条主张·来源·可信度 高/中/低 ｜ 假设 ｜ 待验证 ｜ 方法论增益）。等三者返回，把证据卡抽进研究状态。
2. **选角**：依题型从 5 角色池（严谨 / 务实 / 发散 / 批判 / 证据，见 `roles.md` 选角表）选 3 个，按 spawn 顺序映射甲/乙/丙（拿不准用默认：严谨者 / 发散者 / 批判者；严谨者+批判者尽量在场）。
3. **Round 2（互评轮）**：Claude Code **不能续接已 spawn 的子代理**，所以为每个臭皮匠 **re-spawn 一个新 Opus 子代理**，prompt 注入「它上一轮答案 + 另两家信息（**digest 转自述卡、full 转完整全文**）+ 研究状态 + 选定角色（四段职责）+ 互评议程」。**互评议程**（不复述上轮，只三件事：① 挑另两家隐含假设/边界/反例 ② 把每个争议转成最小可验证动作 ③ 明确哪些已稳可进共识、哪些仍是候选），并更新自述卡与证据卡。三个 re-spawn 同一条消息并发，等返回，更新研究状态。
4. **研究状态机**（主窗口每轮维护）：主张表（每条带 ✅确认/⚔️争议/❌已反驳/🔬待验证 + 支撑证据卡）｜ 主要分歧（≤3，结构化）｜ 证据缺口 ｜ 验证任务（待办/已执行+结果）｜ 已拒方案（防重复讨论）｜ 开放问题。
5. **动态调度**（每轮互评后五选一）：**CONTINUE**（再来一轮，只 re-spawn 与未决分歧最相关的角色、至少 2 个、不让批判职能缺席）/ **CONVERGED**（满足收敛门槛 → 整合）/ **REFOCUS**（偏题 → 下轮注入「回到根问题」，同场累计 2 次升级 ASK_USER）/ **ASK_USER**（根本方向之争、需用户偏好才能定 → `AskUserQuestion`，问题 <20 字、2–4 个互不重叠选项）/ **STALLED**（三人均无实质新增 → 停）。**收敛门槛**含硬条件：≥1 条关键主张有一手来源验证（验证任务非空），否则不算收敛、强制再来一轮。轮数默认 2、最多 3。
6. **整合**：先校准 self-preference（你与臭皮匠同底座、会偏好「像自己」那份，按**证据卡可信度**而非表述流畅度排序，对最认同那份额外证伪），输出 **共识 / 矛盾 / 部分覆盖 / 独特洞见 / 盲点 / 观点演化 / 最终答案**，把已拒方案与验证任务结果纳入审计轨迹，保留分歧不强压共识，注明上下文模式（digest/full）、轮数、本场三角色。

某子代理失败 → 视为该轮缺席，用其余两家继续并标注，绝不当作沉默同意。最终答案用中文。
