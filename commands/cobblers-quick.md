在下面的任务上以 **Quick 模式** 调用 **cobblers** skill（不走多轮 Discuss）。

执行流程：
1. **同一条消息** spawn **3 个独立 Opus 子代理**（`Agent`，`subagent_type: general-purpose`，`run_in_background: true`）。先一句话判题型，给甲/乙/丙各选一行正交「方法论侧重」（第一性原理 / 类比迁移 / 逆向工程，只分叉「怎么推」）。每个 prompt = 用户任务**逐字原文** + 简短指令（用 web/bash 自行研究、作为多位独立专家之一、不会看到他人回答、给完整自洽的中文答案；**若 web/bash 不可用或异常，须显式标「未联网核实」，不得假装查过**）+ 那一行方法论侧重。三份 prompt 主体相同、**不分配角色、不互看**。**prompt 必须自包含**：子代理看不到你俩的对话/更早轮次/上传文件，回答所需的背景（相关代码、先前约束、指代消解）必须显式写进 prompt（背景大可用 `--save` 写 `context.md` 供三家读、收尾出 `transcript.html` 存档，详见 `skills/cobblers/references/discuss.md`）。要求结尾附**自述卡 + 结构化证据卡**（核心主张 ｜ 每条证据标 支持/反驳哪条主张·来源·可信度 高/中/低 ｜ 假设 ｜ 待验证）。
2. 等三者全部返回。任一失败/超时 → 视为缺席，用其余继续并标注，绝不当作沉默同意；不足 2 份则说明并降级。
3. **整合**（不平均、不抹平冲突）：主窗口 Opus 输出 **共识 / 矛盾 / 部分覆盖 / 独特洞见 / 盲点 / 最终答案**。独立同意 = 高置信，**但仅当错误彼此独立**——三家同底座、错误相关，纯记忆推理的一致应判为可能的共享偏置、主动找反例。**校准裁判偏差**：你与三个臭皮匠同源、会偏好「像自己」的那份（self-preference），排序以**证据卡可信度**（一手来源 > 记忆推理）而非表述流畅度为准，对最认同的那份额外证伪。跑过代码或读过一手来源的权重更高。

不要进入互评/多轮（那是 `/cobblers`）。**纯事实 / 强时效 / 已知模型盲区题别用**——三家同底座会齐刷刷错，转 `fusion`（真异构）或外部检索更靠谱。最终答案用中文。
