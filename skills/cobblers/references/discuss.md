# Discuss 模式 —— 3 个臭皮匠多轮讨论

主窗口 Opus（诸葛亮）是唯一的中转枢纽 + 最终整合者。

**关键约束（决定整个实现）**：当前 Claude Code **无法续接一个已 spawn 的子代理**——没有 `SendMessage` 之类的工具，`Agent` 也没有续接参数。子代理一旦返回就结束。所以「多轮讨论」**不是**靠续接同一个子代理，而是靠**每轮重新 spawn 全新子代理**，由主窗口把上一轮的上下文**注入新 prompt** 来重建。主窗口负责保存每轮每家的答案，供下一轮注入。

## 上下文传递模式（可设置）

控制互评轮注入给每个臭皮匠的「另两家信息」有多少：

| 模式 | 注入另两家的 | 完整度 | token | 默认 |
| --- | --- | --- | --- | --- |
| `digest` | 各家自产的**「自述卡」** | 中高（作者本人提炼） | 适中 | ✅ |
| `full` | 各家**完整答案全文** | 高（连论证链都给） | 高 | |

**怎么选**：用户用 `/cobblers --full` 或说「完整上下文」→ `full`；否则 `digest`。把标志从问题文本剥掉再用。

## Round 1 — blind 独立发散

把用户任务**逐字原文** + 简短指令（用 web/bash 自行研究、作为多位独立专家之一、不会看到他人回答、给完整自洽中文答案）构造成**统一 prompt**，三份完全相同，**不分配角色**。要求每家结尾附「自述卡」：

```
【自述卡】
- 核心主张：3–6 条，每条一句话
- 关键证据/来源：标出处（URL/命令/文献）；无一手来源的主张标「未验证」
- 关键假设
- 待验证/不确定
```

**同一条消息 spawn 3 个 Opus 子代理**（`Agent`，`subagent_type: general-purpose`，`run_in_background: true`）。等三者全部返回，**主窗口保存三份答案 + 三张自述卡**（下一轮要注入）。

## Round 2+ — 互评（re-spawn + 注入上下文）

**不续接旧子代理。为每个臭皮匠 re-spawn 一个新 Opus 子代理**（同一条消息并发 3 个），prompt 由主窗口拼装：

```
你是臭皮匠【甲】，正在参加一场多专家讨论的第 N 轮。

你上一轮的立场（完整答案）：
<主窗口注入：甲上一轮的答案>

另外两位专家的观点：
[乙] <按模式注入：乙的自述卡 或 全文>
[丙] <按模式注入：丙的自述卡 或 全文>

当前工作黑板：
<已确认 / 分歧 / 开放问题 / 证据缺口>

你的角色：<严谨者 / 发散者 / 批判者，见 references/roles.md>

互评议程（本轮不复述上轮，只做三件事）：
① 指出乙丙的隐含假设、边界条件、可能反例；
② 把每个争议点转成一个可验证的最小动作（实验/仿真/计算/检索）；
③ 明确哪些结论已足够稳（可进共识）、哪些仍是候选方向。
基于以上更新你的立场：保留/修正/放弃哪些观点 + 为什么，并给出更新后的「自述卡」。
```

角色按 Round 1 的 spawn 顺序固定映射（甲→严谨者、乙→发散者、丙→批判者）。等三者返回，主窗口更新保存的答案与黑板。

## 工作黑板（主窗口每轮维护）

```
- 已确认事实：三家独立认可、经互评仍站得住的点
- 主要分歧：谁 vs 谁、争什么、是「证据可解」还是「需用户定夺」
- 开放问题：尚未解决的子问题
- 证据缺口：哪些主张还缺一手来源
```

## 收敛判断（主窗口当调度器，每轮后）

**满足「收敛门槛」才进整合**，否则再来一轮（re-spawn，只聚焦未决分歧）：
- 已形成明确候选方向；关键分歧已列清（已裁定或已标「需用户/实验定夺」）；证据缺口已标出；每个主要结论有下一步验证路径。

**提前终止**（任一）：
- **stalled**：某轮三人都「无实质新增」→ 停；
- **轮数硬上限**：默认 2 轮（首轮 + 1 互评），最多 3 轮；
- **ASK_USER 暂停**：若核心分歧是**根本方向之争、要靠用户偏好/约束才能定**（非证据可解）→ 用 `AskUserQuestion` 把分歧做成 2–4 个选项暂停问用户，拿到回复再继续或整合。

## 整合（主窗口 Opus，最终交付）

**不强行压成共识，保留分歧。** 给每个关键主张标状态：✅共识 / ⚔️争议 / ❌已反驳 / 🔬待验证。输出：
- **共识** / **矛盾** / **部分覆盖** / **独特洞见** / **盲点**（诸葛亮可自补一条）；
- **观点演化**：关键观点谁首轮提出 → 被谁用什么挑战 → 最终存活/修正/放弃（带状态）；
- **最终答案**：grounded 在以上，先 high-confidence 共识，纳入独特洞见，标注仍不确定处。

呈现时注明上下文模式（digest / full）与轮数。

## 容错与权重

- 某子代理**失败/超时** → 视为**该轮缺席**，用其余两家继续并标注，**绝不当作沉默同意**。
- Round 1 不足 2 家返回 → 降级为可用家数的一次性整合，并说明。
- 证据 > 断言：跑过代码/读过一手来源的，权重高于凭记忆推理的。藏起真冲突比没讨论更糟。
