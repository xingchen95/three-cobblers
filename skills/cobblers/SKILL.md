# 三个臭皮匠（Cobblers）

核心机制是 **独立，再综合**。同一个难题发给 3 个 Opus 子代理，每个都带 web 搜索和 bash、互相看不到对方的工作；然后主窗口 Opus 把三份答案碰撞、整合。多样性靠**独立性**收割：首轮三人拿到主体一字不差的同一任务、**不分配角色**，只额外注入一行「方法论侧重」（第一性原理/类比/逆向…，只分叉「怎么推」不分叉「推向哪」），在不污染独立性的前提下把发散从「同一吸引盆里的温度抖动」推到「不同推理路径」。角色要到互评轮才登场。

**一条硬规则：主窗口 Opus 永远是「诸葛亮」（裁判 + 整合者），流水线不可逆。** 臭皮匠子代理无法回头 spawn 主窗口，所以整合永远在主窗口发生——真正理由是**可见性**（只有它能看到全部答案）。注意 Discuss 模式下主窗口还维护黑板、判收敛、贴标签，是**掌握议程的主编**而非纯旁观者；且它与三个臭皮匠**同底座**，会本能偏好「像自己」的那份（self-preference）——整合时按证据强度而非表述流畅度排序，对最认同的那份额外证伪。

## 选模式

| 模式 | 命令 | 流程 | 何时用 |
| --- | --- | --- | --- |
| **Discuss（主打）** | `/cobblers` | 首轮 blind 独立 → 多轮互评（加角色）→ 整合 | 真正需要来回碰撞、互相挑错的难题 |
| **Quick** | `/cobblers-quick` | 3 个 Opus 一次性 blind → 直接整合 | 想要多视角但不必多轮，快且省 |

默认路由：用户提到「讨论 / 辩论 / 多轮 / 互相」→ **Discuss**；只想「多个意见 / cross-check / 几个模型看看」→ **Quick**；拿不准 → **Discuss**。用户用了 `/cobblers` 或 `/cobblers-quick` 就按命令来。

**⚠ 何时别用 cobblers（硬路由）**：三个臭皮匠是同底座 Opus、错误相关，对**纯事实查证 / 强时效（价格、版本、最新进展）/ 已知落在模型盲区**的问题会**齐刷刷错向同一处**，而精美的审计轨迹反而抬高你对错误的信心。这类题**别用 cobblers 包装同源答案**——主动转 **fusion**（Opus+GPT+Gemini 真异构 panel）或外部检索。cobblers 的主场是**推理 / 设计 / 调试 / 权衡**这类「多视角碰撞有增量」的题。

## Discuss 模式

照 `references/discuss.md` 执行：首轮 blind 独立（方法论分叉 + 结构化证据卡）→ 主窗口从 **5 角色池按题型选 3**（拿不准就用默认 严谨/发散/批判）→ **互评轮注入**（启用 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 则用 `SendMessage` 续接同一子代理、保留其完整推理；否则 re-spawn 重建）→ 维护**研究状态**（轻量优先，难题才上完整状态机）→ **动态调度器**每轮判 CONTINUE/CONVERGED/REFOCUS/ASK_USER/STALLED → 整合。角色池与选角表见 `references/roles.md`。加 `--save` 可把全程写进会话工作目录（`context.md`/`state.md`/`transcript.md`）并渲染 `transcript.html` 存档——只主窗口写、子代理只读，详见 `references/discuss.md`「会话工作目录」。

**上下文传递模式（可设置）**：互评轮另两家的信息怎么给——**`full`（默认）**转发各家完整 Round 1 全文（连论证链都给，挑隐含假设最扎实），或 `digest` 转发各家「自述卡」（核心主张+证据卡+假设，省 token）。cobblers 主打高风险题，默认就给全文；用户用 `/cobblers --digest`、或说「精简上下文」→ `digest`（低风险 / 答案过长怕爆上下文时）。详见 `references/discuss.md`。

## Quick 模式

1. **同一条消息** spawn 3 个 Opus 子代理（`Agent` 工具，`subagent_type: general-purpose`，`run_in_background: true`）。每个 prompt = 用户任务**逐字原文** + 简短指令：用 web/bash 自行研究、作为多位独立专家之一、不会看到他人回答、给一份完整自洽的中文答案。三份 prompt 主体相同、**不分配角色**，只各注入一行「方法论侧重」（第一性原理/类比/逆向，按题型选；只分叉怎么推）。要求结尾附结构化**自述卡 + 证据卡**（核心主张 ｜ 每条证据标 支持/反驳哪条主张·来源·可信度 高/中/低 ｜ 假设 ｜ 待验证）。
2. 等三者全部返回。任一失败/超时 → 视为**缺席**，用其余继续，**绝不当作沉默同意**；不足 2 份则说明并降级。
3. 主窗口整合（不做平均、不抹平冲突）：**共识 / 矛盾 / 部分覆盖 / 独特洞见 / 盲点 / 最终答案**。独立同意 = 高置信，**但仅当三家的错误彼此独立时**——三个臭皮匠同底座、错误相关，若一致结论只靠「记忆推理」而无一手验证，应判为**可能是共享偏置**、主动找反例而非升置信。跑过代码或读过一手来源的，权重高于凭记忆推理的。

## 通用规则

- 子代理之间**不能直接通信**——任何「互看」必须由主窗口中转。
- **首轮永远 blind**：构造 prompt 时不要预先消化/总结任务，原样传递；不要分配角色。
- **Prompt 自包含（子代理对话失明）**：子代理只看得到你给它的那条 prompt，看不到你俩的对话历史 / 更早轮次 / 上传的文件。所以「逐字传问题」之外，**回答所需的背景（相关代码、先前约束、指代消解）必须由主窗口显式注入**；背景大时用 `--save` 的 `context.md` 承载（见 `references/discuss.md`）。
- 最终答案默认用**中文**（除非用户要求其他语言）。
- **前提**：会话模型为 Opus 4.8；子代理与裁判都继承会话模型。换成别的模型，"Opus" 只是名义。
- **呈现**：先给最终答案，再给审计轨迹（五段分析；Discuss 多一节「观点演化」）。注明跑的是 Discuss 还是 Quick、几个臭皮匠参与、有无缺席。
- **证据接地（mindforge 式）**：每家的关键证据走结构化「证据卡」（支持/反驳哪条主张 · 来源 · 可信度 高/中/低）；整合与收敛按证据卡可信度排序，纯记忆级证据的一致视为可能的共享偏置而非高置信。**这是真正的去相关杠杆**——同底座的角色扮演只造视角张力、不造误差独立，别指望它。
- **轻量优先（防过度工程）**：有牙的核心是 证据卡 + 收敛硬门槛 + REFOCUS + ASK_USER，这些留死；**5 角色精细选角、完整研究状态机是难题才上的升级档**，简单问题别堆——纯 prompt 没强制力，机制越精密被真实执行的比例越低。
- **成本**：Quick ≈ 3× 单次；Discuss ≈ 3–5×（动态调度器可省下「无关角色」的 re-spawn）。低风险/简单问题直接单模型答更划算——别为了用而用。
